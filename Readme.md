# Asterisk IVR — Docker

> Голосовое IVR-меню на базе Asterisk 18, упакованное в Docker-контейнер.  
> Русскоязычные аудиофайлы генерируются автоматически через gTTS при старте.

---

## Структура проекта

```
.
├── config/
│   ├── sip.conf          # SIP-аккаунты и NAT-настройки
│   ├── extensions.conf   # Диалплан IVR
│   ├── modules.conf      # Явная загрузка модулей (autoload=no)
│   └── rtp.conf          # Диапазон RTP-портов
├── scripts/
│   ├── entrypoint.sh     # Точка входа контейнера
│   └── gen_sounds.py     # Генерация WAV/ULAW через gTTS + sox
├── Dockerfile
├── docker-compose.yml
└── README.md
```

## Быстрый старт

```bash
# Клонировать репозиторий
git clone https://github.com/insanepadawan/sip-module.git
cd asterisk-ivr

# Собрать и запустить
docker compose up --build
# или
docker build -t asterisk-ivr .
```

При первом запуске контейнер автоматически:
1. Определяет внешний IP и прописывает его в `sip.conf`
2. Генерирует все русскоязычные аудиофайлы через gTTS
3. Запускает `tcpdump` для захвата SIP-трафика
4. Поднимает Asterisk

---

## Как подключиться

Используйте любой SIP-клиент (Linphone, Zoiper, MicroSIP и др.):

| Параметр           | Значение              |
|--------------------|-----------------------|
| Сервер             | `<IP вашего хоста>`   |
| Порт               | `5060` (UDP)          |
| Логин              | `1001`, `1002`, `1003`|
| Пароль (временный) | `secret123`           |

После регистрации наберите **100**, чтобы попасть в IVR-меню.

## 🎛Сценарий IVR

```
Вызов на номер 100
│
├─ Нажмите 1 → «Вы улыбнулись» → завершение вызова
│
├─ Нажмите 2 → «Нажмите любую цифру от 0 до 9»
│              → «Вы нажали цифру X» → завершение вызова
│
├─ Нажмите 3 → запись голосового сообщения (до 60 сек, стоп — #)
│              → воспроизведение записи обратно → завершение вызова
│
└─ Таймаут / неверная клавиша → повтор главного меню
```

## Генерация звуков

Скрипт `gen_sounds.py` при старте создаёт файлы в `/var/lib/asterisk/sounds/ivr/`:

- **gTTS** — синтезирует русскоязычные фразы в MP3
- **sox** — конвертирует в WAV и ULAW (8000 Гц, моно) для Asterisk
- **Бип** — генерируется напрямую через `sox synth` (без TTS)
- Уже существующие файлы пропускаются — повторный старт не пересоздаёт их

## Сетевой режим

Контейнер использует `network_mode: host` — это необходимо для корректной работы SIP NAT и RTP-медиапотоков.

| Порт           | Протокол | Назначение        |
|----------------|----------|-------------------|
| `5060`         | UDP      | SIP-сигнализация  |
| `10000–10100`  | UDP      | RTP-медиапотоки   |

---

## Захват трафика

При старте автоматически запускается `tcpdump`, который пишет SIP и RTP в файл:

```
/var/log/asterisk/sip_capture.pcap
```

Открыть в Wireshark для анализа:

```bash
# Скопировать из контейнера
docker cp asterisk-ivr:/var/log/asterisk/sip_capture.pcap ./sip_capture.pcap

# Открыть
wireshark sip_capture.pcap
```

## Полезные команды

```bash
# Логи в реальном времени
docker compose logs -f

# Консоль Asterisk внутри контейнера
docker exec -it asterisk-ivr asterisk -r

# Список зарегистрированных SIP-пиров
docker exec -it asterisk-ivr asterisk -rx "sip show peers"

# Активные вызовы
docker exec -it asterisk-ivr asterisk -rx "core show channels"

# Перезагрузить диалплан без рестарта
docker exec -it asterisk-ivr asterisk -rx "dialplan reload"
```

## Требования

- Docker ≥ 20.10
- Docker Compose ≥ 2.0
- Интернет-доступ при первом запуске (для определения внешнего IP и загрузки gTTS)
- Права `NET_ADMIN` и `SYS_PTRACE` (для tcpdump — только в dev-окружении)

## Важное замечание

- Пароли SIP-аккаунтов (`secret123`) предназначены **только для тестирования** — смените перед деплоем
